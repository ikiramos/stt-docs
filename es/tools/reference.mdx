Todas las herramientas son de **solo lectura** salvo las dos de *Acciones*, que gastan créditos de la cuenta y siempre piden confirmación. Los resultados son JSON compacto pensado para ventanas de contexto de IA (las listas devuelven máximo 25 filas). Las descripciones se muestran en inglés: son las que lee tu asistente, generadas desde el servidor.

## Productos

### `search_products`

Search winning e-commerce/dropshipping products tracked from the Facebook Ad Library. Filter by free-text search (product name, store domain, advertiser), target countries (ISO-2), niche (2-level taxonomy: vertical or exact 'Vertical &gt; Sub' leaf), ad-copy language, minimum active ads and days running. Sort by opportunity score (default), ad-count growth, estimated ad spend or longevity. Returns compact product cards with id (use get_product for full detail). Cards include myshopifyDomain, two store domains sharing it are the SAME operator (twin stores).

### `trending_products`

The products gaining active ads the FASTEST right now (7-day ad-count growth, minimum scale filter). Zero-config shortcut for 'what is trending / heating up today'.

| Parámetro | Tipo | Obligatorio | Notas |
|---|---|---|---|
| `limit` | integer | No | Default: `10`. Range: 1–25 |

### `get_product`

Full detail for ONE product by its id (from search_products/trending_products): metrics, score, country distribution, weekly ad-activity history and its top ads (creative URLs + copy).

| Parámetro | Tipo | Obligatorio | Notas |
|---|---|---|---|
| `productId` | string | Yes | — |

### `get_product_sources`

PROVENANCE / funnel strategy of a product's ads: which Facebook pages advertise it and from which store domains, with per-page ad volumes (sourceBreakdown), plus its 'door map': every distinct landing URL its ads point at, typed (product page / advertorial / lander / quiz) with volumes (landingUrls). Answers 'HOW is this product being sold and by whom', key piece when reconstructing a competitor's selling strategy.

| Parámetro | Tipo | Obligatorio | Notas |
|---|---|---|---|
| `productId` | string | Yes | — |

### `get_product_history`

A product's weekly ad-activity time series PARTITIONED per advertising Facebook page (who ramped up, who dropped off, when each page joined). Sum partitions for the total.

| Parámetro | Tipo | Obligatorio | Notas |
|---|---|---|---|
| `productId` | string | Yes | — |

## Marcas

### `get_brand`

Analyze / spy on a BRAND, STORE or COMPETITOR by its domain (e.g. 'lymphoria.co'), the starting point to reconstruct a competitor's selling strategy: every Facebook fan page it advertises through (many brands run 10-20 'satellite' pages), total active ads, detected tech stack (Shopify/funnel builders) and ad pixels. The competitor X-ray. Chain with search_products(search=domain) and get_product_sources to map its products and funnels.

| Parámetro | Tipo | Obligatorio | Notas |
|---|---|---|---|
| `domain` | string | Yes | Store domain or myshopify domain |

### `search_brands`

Search tracked BRANDS (stores grouped across all their fan pages). Filter by domain text, detected tech (e.g. ['shopify','funnelish','clickfunnels','gempages','replo']), ad pixels (e.g. ['meta','tiktok','snapchat','google']), dominant store language or minimum active ads. Sorted by active ads. Great for stack espionage: 'brands running funnelish with a TikTok pixel'.

| Parámetro | Tipo | Obligatorio | Notas |
|---|---|---|---|
| `q` | string | No | Domain contains… |
| `tech` | array of string | No | — |
| `pixels` | array of string | No | — |
| `language` | string | No | ISO 639-1 dominant language of the brand's store |
| `minActiveAds` | integer | No | — |
| `limit` | integer | No | Default: `10`. Range: 1–25 |

### `get_domain_intel`

Tech stack + ad pixels detected on a competitor store DOMAIN (even if it's not a tracked brand yet): platform (Shopify/Woo/funnel builders like ClickFunnels, Funnelish, GemPages, Replo) and pixels (Meta, TikTok, Google, GTM, Snapchat, Pinterest…).

| Parámetro | Tipo | Obligatorio | Notas |
|---|---|---|---|
| `domain` | string | Yes | — |

## Acciones (cuestan créditos)

### `request_brand_scan`

LAUNCH a live scan of a brand's Facebook Ad Library presence (scrapes all its current ads, links products, computes metrics, takes a few minutes). COSTS 300 CREDITS from the user's account, like the in-app 'brand request'. Provide EITHER a Facebook Ad Library URL or a Facebook page id. Returns a jobId, poll get_brand_scan_status until finished, then use get_brand / search_products on the fresh data. Ask the user for confirmation before calling this.

| Parámetro | Tipo | Obligatorio | Notas |
|---|---|---|---|
| `adLibraryUrl` | string | No | https://www.facebook.com/ads/library/?...view_all_page_id=… |
| `facebookPageId` | string | No | Numeric FB page id, the URL is built for you |

### `get_brand_scan_status`

Status of a brand scan launched with request_brand_scan (pending → processing → completed/failed).

| Parámetro | Tipo | Obligatorio | Notas |
|---|---|---|---|
| `jobId` | string | Yes | — |
